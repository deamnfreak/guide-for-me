// phase one — html
01 The Skeleton — Always This Order

Every HTML file has the same bones. Always. Memorize this structure before anything else because everything else builds on top of it.

<!DOCTYPE html> ← not a tag. just tells browser: this is HTML5. <html lang="en"> ← root element. everything lives inside this. <head> ← invisible settings area. nothing here shows on page. <meta charset="UTF-8"> ← enables emojis and special characters. always include. <meta name="viewport" content="width=device-width, initial-scale=1.0"> ← makes page look right on phones. always include. <title>page title</title> ← text in the browser tab. <link href="fonts url" rel="stylesheet"> ← google fonts goes here. <style> /* ALL your CSS goes in here */ /* but write your HTML first before touching this */ </style> </head> <body> ← everything VISIBLE goes here. this is where you work. <!-- your html structure --> <script> ← JavaScript. ALWAYS last, right before </body>. </script> </body> </html>
why script goes last JS often needs to find HTML elements to do things to them. if JS runs before the HTML is loaded, it finds nothing and breaks. putting it last means everything is already there when JS runs.
comments in HTML vs CSS HTML comment: <!-- this is a comment -->
CSS comment: /* this is a comment */

they're different. using CSS comment syntax in HTML = broken. don't mix them.
02 Tags — What They Are & What They Mean

Tags label what something IS. Most come in pairs — opening and closing. The closing one has a / before the name.

<div>content here</div> ← has content inside. needs closing tag. <p>a paragraph</p> <h1>big heading</h1> <input> ← self-closing. no content inside. no closing tag. <img src="photo.png" alt="desc"> <br>

Every tag you'll actually use:

<div>a generic box. no visual meaning on its own. just groups things so you can style them together. use for most containers.
<span>like a div but INLINE — wraps words inside a sentence without breaking to a new line. use for styling one word or phrase differently.
<p>a paragraph. adds spacing above and below automatically. use for blocks of readable text.
<h1> — <h6>headings. h1 = biggest. h6 = smallest. use for page titles and section headers. only one h1 per page.
<a>a link. needs href="" to work. <a href="https://chub.ai">click</a>
<ul> + <li>unordered list. ul = the whole list. li = each item inside it. <ol> = same but numbered.
<nav>navigation bar. works like a div but signals "this is a nav." good practice to use it.
<form>container for input fields. handles submission. give it id="" so JS can find it.
<input>single-line text box. self-closing. needs type="" and name="" (name must match database column).
<textarea>multi-line text box. has open and close tags. user can resize it vertically.
<select>dropdown menu. each choice is an <option> inside it.
<button>clickable button. use type="submit" inside a form to make it submit.
<img>image. self-closing. src="" = url or file path. alt="" = description for screen readers.
<footer>the bottom of the page. works like div but signals "this is a footer."
div vs span — the one thing to actually memorize div = BLOCK. takes up full width. starts on its own line.
span = INLINE. stays inside the text flow. no line break.

use div for containers and sections.
use span when you want to style a word inside a sentence.

Span example:

<p>I make <span class="highlight">dark content</span> for adults.</p>
result once CSS is added

I make dark content for adults.

Only "dark content" gets the color. The rest is untouched. The line doesn't break.

03 Attributes — The Extra Info Inside Tags

Attributes go INSIDE the opening tag and give extra information. Format is always name="value".

<div class="hero"> ↑ class = how CSS targets this element. CSS uses dot: .hero { } <div id="s-rules"> ↑ id = unique name. one per page. CSS uses hash: #s-rules { } also used for nav anchor links: href="#s-rules" <a href="https://chub.ai">visit chub</a> ↑ href = where the link goes <img src="photo.png" alt="my photo"> ↑ src = file or url. alt = description. <input type="text" name="requester_name" required> ↑ type = kind of input. name = database column name. required = must fill before submit.
class vs id class = reusable. many elements can share the same class.
id = unique. only one element per page should have each id.

multiple classes: class="tag accent" — space separated in HTML, chained in CSS: .tag.accent { }
anchor link: href="#s-rules" scrolls to id="s-rules". they must match exactly.
04 Structure & Nesting

HTML is nested — elements live INSIDE other elements. Like folders inside folders. Indentation shows what's inside what.

<div class="wrapper"> ← outer box <div class="section"> ← inside wrapper <h2>Title</h2> ← inside section <p>Text here.</p> ← inside section <ul> ← list inside section <li>item one</li> ← item inside list <li>item two</li> </ul> </div> ← closes section </div> ← closes wrapper
the golden rule — never break this open and close in the right order.

✓ <div><p>text</p></div> — p opens and closes before div closes
✗ <div><p>text</div></p> — closing order is wrong

wrong nesting = layout breaks in silent, confusing ways.

Think in boxes BEFORE writing code. Draw it out:

page ├── nav bar ├── hero section │ ├── heading │ ├── subtext │ └── tags row └── main wrapper ├── section A — what i make │ ├── title row (A. + "What I Make") │ └── two column grid │ ├── yes box → list items │ └── no box → list items ├── section B — rules │ ├── title row │ └── numbered list ├── section C — form └── section D — queue

Build the two-column layout in HTML first (it'll look ugly — that's fine):

<div class="yn-grid"> <div class="yn-box yes"> <div class="yn-title">✓ Will Do</div> <ul class="yn-list"> <li>Original characters</li> <li>Dark / taboo dynamics</li> </ul> </div> <div class="yn-box no"> <div class="yn-title">✗ Won't Do</div> <ul class="yn-list"> <li>Minors in any context</li> </ul> </div> </div> ↑ write this, save it, look at it ugly, then move to CSS to fix it.
05 Forms — All The Pieces
<form id="req-form"> ← id so JS can find it. don't change unless u update JS too. <div class="form-group"> ← wrapper for one field + its label <label>Your username</label> <input type="text" name="requester_name" ← must match supabase column name exactly placeholder="@username" ← hint text, disappears when typing starts required> ← can't submit without filling this </div> <div class="form-group"> <label>Request type</label> <select name="type" required> <option value="">— select —</option> ← placeholder. value="" = required won't accept it. <option value="bot-card">Bot Card</option> ← value = what gets saved. text = what user sees. <option value="lorebook">Lorebook</option> </select> </div> <div class="form-group"> <label>Character concept</label> <textarea name="concept" placeholder="who is this character..." required></textarea> ← textarea needs both open AND close tag </div> <button type="submit">Submit →</button> </form>

06 How CSS Works

CSS is written in rules. Each rule has a selector (what to target) and declarations (what to change). All CSS lives inside <style></style> in your head section.

selector { property: value; property: value; } .hero { background: #000000; color: #ffffff; padding: 40px; text-align: center; }

How to target things:

.classname { }
all elements with class="classname"
#idname { }
the one element with id="idname"
div { }
ALL div elements (use carefully)
.parent .child { }
.child only when inside .parent
.box.active { }
elements with BOTH classes (no space)
.button:hover { }
when mouse is over .button
.item::before { }
fake element inserted before .item's content
body:has(.x[open]) .nav { }
.nav when .x[open] exists anywhere in body — your toggle uses this
07 CSS Variables — Write These First

Before any other CSS, define your colors and fonts as variables in :root. Change a color once, updates everywhere.

:root { --bg: #0a0a0a; --surface: #111111; --border: #2a2a2a; --purple: #bc00ff; --pink: #e82fdc; --text: #d0d0d0; --dim: #d0d0d0; --mono: 'Share Tech Mono', monospace; } use them anywhere: .hero { background: var(--bg); color: var(--text); border: 1px solid var(--border); }
why this matters without variables: change accent color = find and replace 40+ hex codes.
with variables: change --purple once = done everywhere. ♡

09 The Box Model

Every element is a box with four layers. This explains why things are spaced the way they are.

MARGIN — space outside, pushes other elements away
BORDER — the visible edge
PADDING — space inside, between border and content
CONTENT
padding vs margin in one sentence each padding = inside. like foam protecting something inside a box.
margin = outside. like the gap between the box and the wall.
padding: 20px; ← all 4 sides padding: 20px 40px; ← top+bottom 20 / left+right 40 padding: 10px 20px 30px 40px; ← top right bottom left (clockwise) margin: 0 auto; ← 0 top/bottom, auto sides = centers the element
10 Display, Flex & Grid

display controls how children are laid out inside an element.

display: block — default for div, p, h1. full width. own line.

display: flex — children side by side in a row. most used.

display: grid — children in rows AND columns.

display: none — completely hidden. JS uses this to show/hide things.

Flexbox:

.container { display: flex; flex-direction: row; ← side by side (default) flex-direction: column; ← stacked top to bottom align-items: center; ← vertical center (when row) justify-content: center; ← horizontal center justify-content: space-between; ← push to opposite ends gap: 16px; ← space between children flex-wrap: wrap; ← wrap to next line if too wide }

Grid:

.container { display: grid; grid-template-columns: 1fr 1fr; ← 2 equal columns grid-template-columns: 1fr 1fr 1fr; ← 3 equal columns grid-template-columns: 2fr 1fr; ← 66% / 33% grid-template-columns: repeat(3, 1fr); ← same as 1fr 1fr 1fr gap: 16px; } 1fr = 1 fraction of available space. 1fr 1fr = two equal halves. 2fr 1fr = left gets 2/3, right gets 1/3.
flex vs grid flex = one direction. row of links, column of form fields, center one thing.
grid = two directions. side-by-side boxes, card layouts.

12 Pseudo Elements

Fake elements CSS generates. Don't exist in HTML but render on page. Used for decorative things, custom symbols, overlays.

.element::before { content: ''; ← REQUIRED. always. even if empty string. without this = invisible. display: block; background: red; } .element::after { content: '→'; ← can have text or symbols color: var(--purple); }
the most common mistake forgetting content: ''
no error. just silently doesn't appear.
first thing to check when a ::before or ::after isn't showing up.

15 Colors — hex, rgb, rgba

Three formats. You'll use all three.

HEX — most common color: #bc00ff; color: #fff; ← shorthand: #fff = #ffffff RGB color: rgb(188, 0, 255); RGBA — rgb + alpha (opacity). 0 = invisible, 1 = solid. background: rgba(188, 0, 255, 0.1); ← 10% purple tint background: rgba(0, 0, 0, 0.5); ← 50% black overlay background: rgba(255, 255, 255, 0.08); ← barely-there white
when to use rgba anytime you want a color but semi-transparent — tinted card backgrounds, soft glows, overlays. hex doesn't support transparency. rgba does.

16 Transitions & Animations

Transitions — smoothly animate a property when it changes (like on hover):

.button { background: #000; transition: background 0.2s ease, color 0.2s ease; ↑ format: property duration timing-function } .button:hover { background: var(--purple); ↑ transition makes this animate smoothly instead of snapping } timing functions: ease ← slow, fast, slow (default, feels natural) linear ← constant speed (use for spinning borders) ease-in-out ← slow, fast, slow (good for pulsing)

Animations — for things that animate on their own:

step 1: define what happens at each point @keyframes pulse { 0%, 100% { opacity: 1; } 50% { opacity: 0.4; } } @keyframes spin { to { transform: rotate(360deg); } } step 2: apply it .dot { animation: pulse 2s ease-in-out infinite; ↑name duration timing how many times }
transition vs animation transition = reacts to something (hover, JS toggle, class change).
animation = runs on its own continuously. defined with @keyframes.
17 Media Queries — Mobile

Apply different CSS at different screen sizes. How you stop things looking broken on phones.

@media (max-width: 600px) { /* only applies on screens 600px or smaller */ .yn-grid { grid-template-columns: 1fr; ← stack to one column on phone } .hero h1 { font-size: 28px; ← smaller heading on phone } } common breakpoints: max-width: 480px ← small phones max-width: 768px ← tablets max-width: 992px ← small laptops (chub uses this)
the workflow write your CSS for desktop first. look at it on a phone (or resize browser).
whatever looks broken — fix it inside a max-width media query.

20 Gradients

Three types. You've already used all of them.

LINEAR — straight line between colors background: linear-gradient(90deg, #bc00ff, #e82fdc); ↑ 90deg = left to right. 0deg = bottom to top. 45deg = diagonal. background: linear-gradient(135deg, #000 45%, #1a0033); ↑ adding % = hard stop. no fade. black until 45% then switch. RADIAL — radiates out from a center point background: radial-gradient(ellipse at 50% 0%, rgba(188,0,255,0.15), transparent 60%); ↑ shape, position, colors. transparent 60% = fades to nothing by 60%. 21 Specificity & !important

When two CSS rules target the same element, the browser has to decide which one wins. That decision is called specificity. Understanding this explains why your styles sometimes get ignored.

specificity hierarchy — higher number wins: inline style (style="" on the element) = 1000 points #id selector = 100 points .class selector / :pseudo-class = 10 points element selector (div, p, h1) = 1 point examples: div p { color: red; } ← 2 points (div=1 + p=1) .section p { color: blue; } ← 11 points (.section=10 + p=1) — this wins #hero p { color: green; } ← 101 points — this wins over both when two rules have equal specificity, the LAST one in the file wins.
!important — the nuclear option adding !important to a declaration overrides ALL specificity rules.
nothing can override it except another !important with higher specificity.

color: #bc00ff !important;

this is why EVERYTHING in your Chub bio needs !important — Chub's own CSS has high specificity selectors that would override your styles otherwise. !important is the only way to guarantee your rule wins on someone else's platform.

in your OWN sites (GitHub Pages, Nekoweb) you generally DON'T need !important because you control all the CSS. only use it when fighting platform styles.
if your style isn't applying, check DevTools — crossed-out CSS = being overridden by something more specific. solutions in order of preference: 1. make your selector more specific: .section .title { } instead of .title { } 2. add the parent element: nav .nav-logo { } instead of .nav-logo { } 3. last resort: add !important color: red !important;
22 Transform

The transform property moves, scales, rotates, or skews elements. It doesn't affect page layout — the element stays in its original space, it just visually changes.

TRANSLATE — move the element transform: translateX(42px); ← move right 42px (negative = left) transform: translateY(-50%); ← move up by 50% of element height transform: translate(42px, -50%); ← move right AND up at the same time transform: translateX(-50%); ← classic centering trick ROTATE transform: rotate(45deg); ← 45 degree clockwise rotation transform: rotate(-90deg); ← counterclockwise transform: rotate(180deg); ← upside down SCALE — resize transform: scale(1.05); ← 5% bigger (good for hover effect) transform: scale(0.9); ← 10% smaller transform: scaleX(-1); ← flip horizontally COMBINING — multiple transforms in one line transform: translate(42px, -50%); ← your toggle knob uses this transform: translateY(-4px) scale(1.05); ← lift + enlarge on hover
the centering trick classic way to center an absolutely positioned element:

position: absolute;
left: 50%;
transform: translateX(-50%);

left: 50% puts the LEFT EDGE at center. translateX(-50%) then shifts it back by half its own width. net result = perfectly centered. you've seen this pattern in the request site queue table.
23 Filter & Backdrop-Filter

filter applies visual effects to an element. backdrop-filter applies them to whatever is BEHIND the element.

FILTER — applied to the element itself filter: blur(4px); ← blurs the element filter: brightness(0.8); ← 80% brightness (darker) filter: brightness(1.2); ← 120% brightness (lighter) filter: saturate(1.5); ← 150% saturation (more vivid) filter: contrast(1.1); ← 110% contrast filter: invert(1); ← fully inverts colors filter: grayscale(1); ← black and white filter: drop-shadow(0 4px 12px rgba(0,0,0,0.5)); ← shadow (works on transparent PNGs) COMBINING filters: filter: blur(4px) saturate(1.15) contrast(1.05); ← your banner image uses this RECOLORING with filter — what you used on the sun/moon emoji: filter: invert(1) sepia(1) saturate(10) hue-rotate(270deg) brightness(1.1); ↑ this chain shifts the color. hue-rotate changes the hue angle. use cssgenerator.org/filter to generate these values from a hex color. BACKDROP-FILTER — applied to what's BEHIND the element (glass effect) backdrop-filter: blur(12px); backdrop-filter: blur(12px) saturate(180%); ↑ your toggle button uses this. makes it look frosted/glassy. the element needs a semi-transparent background to show the effect.
filter vs backdrop-filter filter: blur() blurs the element itself and everything inside it.
backdrop-filter: blur() blurs what's behind it, like frosted glass.

backdrop-filter needs: background with some transparency (rgba) to work visibly. a fully opaque background blocks the effect.
24 The details & summary Toggle Trick

The <details> and <summary> HTML elements give you a native browser toggle with NO JavaScript. This is the technique behind your Chub dark/light mode toggle.

basic usage: <details> <summary>click me</summary> <p>this content shows when open</p> </details> how it works: — closed by default — clicking summary toggles open/closed — when open, the [open] attribute appears on the details element — you can target [open] in CSS targeting open/closed state in CSS: details[open] summary { color: var(--purple); ← when open } details summary { color: #bbb; ← when closed } the body:has() trick — affect OTHER elements based on toggle state: body:has(details[open]) .nav { background: white; ← nav turns white when details is open } body:has(.theme-toggle[open])::before { background: #e82fdc; ← your light mode bg swap }
why this matters for Chub and J.AI script tags don't work on Chub/J.AI profiles — JavaScript is blocked.
details + body:has() = a pure CSS toggle with no JS required.

this is what you cracked that people said wasn't possible. ♡

the invisible details trick: make details opacity: 0 and position it over a visible div. clicking the invisible details toggles the state. body:has() then updates the visible div. that's your toggle button.
hide the default "▶ Details" arrow: summary { list-style: none; } summary::-webkit-details-marker { display: none; } open by default (add the open attribute in HTML): <details open> <summary>...</summary> </details>
25 Overflow & Scroll Boxes

What happens when content is bigger than its container. You'll need this for scroll boxes, hidden overflow on cards, and text that shouldn't wrap.

overflow values: overflow: visible; ← default. content spills outside the box. overflow: hidden; ← clips anything outside. nothing scrolls. overflow: auto; ← adds scrollbar only when content overflows. overflow: scroll; ← always shows scrollbar even if not needed. control X and Y separately: overflow-x: hidden; ← hide horizontal overflow (stops side scroll) overflow-y: auto; ← scroll vertically when needed making a scroll box: .scroll-box { max-height: 300px; ← caps the height overflow-y: auto; ← adds vertical scroll when content exceeds 300px } hide the scrollbar visually but keep scroll working: .scroll-box::-webkit-scrollbar { display: none; } .scroll-box { -ms-overflow-style: none; scrollbar-width: none; }
border-radius + overflow: hidden trick if you have a rounded container with an image inside and the image ignores the border-radius (corners still sharp):

add overflow: hidden to the container.

the overflow clips the image to the rounded shape. ♡
26 CSS Selectors — The Ones You'll Actually Use

Beyond the basics. These are the selectors that make things feel smart.

DESCENDANT — element anywhere inside parent .section p { } ← any p inside .section at any depth DIRECT CHILD — only immediate children .section > p { } ← only p directly inside .section, not nested deeper ADJACENT SIBLING — element immediately after another h2 + p { } ← p that comes right after an h2 NTH-CHILD — target specific positions in a list li:nth-child(1) { } ← first list item li:nth-child(2) { } ← second list item li:nth-child(odd) { } ← every odd item (1, 3, 5...) li:nth-child(even) { } ← every even item (2, 4, 6...) NOT — exclude something li:not(:last-child) { } ← all list items except the last one .box:not(.active) { } ← all .box elements that don't have .active HAS — parent based on what it contains (you use this for your toggle) body:has(.toggle[open]) .nav { } ← .nav when toggle is open div:has(> img) { } ← div that directly contains an img ATTRIBUTE SELECTORS [open] { } ← any element with the open attribute [type="text"] { } ← inputs with type="text" [class*="ant-col"] { } ← elements whose class CONTAINS "ant-col" — your Chub selectors use this
27 Fonts — Loading, Using, Fallbacks

Everything about fonts you need to know.

GOOGLE FONTS — easiest method <link href="https://fonts.googleapis.com/css2?family=Rajdhani:wght@700;900&display=swap" rel="stylesheet"> then use it: font-family: 'Rajdhani', sans-serif; loading multiple fonts in one link tag (more efficient): https://fonts.googleapis.com/css2?family=Rajdhani:wght@700;900&family=Share+Tech+Mono&display=swap SELF-HOSTED FONT (from dafont etc) — upload file to your repo @font-face { font-family: 'MyFont'; ← the name YOU give it src: url('myfont.woff2') format('woff2'), url('myfont.ttf') format('truetype'); font-weight: normal; } then use it exactly like Google Fonts: font-family: 'MyFont', sans-serif; FONT STACK — always include a fallback font-family: 'Share Tech Mono', 'Courier New', monospace; ↑ tries Share Tech Mono first. if not loaded, tries Courier New. if that fails, any monospace. generic fallback families: serif ← has serifs (Times New Roman style) sans-serif ← no serifs (clean, modern) monospace ← fixed width, code-style
font-display: swap the display=swap in Google Fonts URLs is important.
without it: page shows blank text until font loads.
with swap: shows fallback font immediately, swaps to custom font when ready.

always include it. Google Fonts adds it automatically when you use their embed code.
28 The Stuff No One Tells You

Observations from actually building things. Not rules — just true things.

on learning you will forget syntax constantly. that's normal. that never fully stops.
the goal isn't to memorize everything. it's to know what exists so you know what to look up. that's what separates someone who codes from someone who doesn't — not memorization, but familiarity.

you're already building that familiarity. every time you read a CSS property and go "oh that's what that does" — that's learning happening. ♡
on using AI as a tool using AI to help you code is like using Google — it's a tool, not a crutch, as long as you're trying to understand what comes out.

the difference between a vibecoder and someone learning: vibecoder copies and pastes without reading it.
you're learning: you read it, question it, catch mistakes, ask why.

that's the whole difference. and you've been doing the right one. ♡
on debugging "it's not working" is never the whole problem statement.
train yourself to say: "I expected X, but I'm getting Y, and I've tried Z."

that's how you ask better questions — to me, to Google, to yourself. the more specific you can describe the problem, the faster it gets solved.
on CSS specificity fights (especially on Chub) when your style isn't applying and you don't know why:
1. open DevTools
2. find your rule
3. if it's crossed out = something more specific is overriding it
4. either make your selector more specific OR add !important

on platforms you don't control (Chub, J.AI): !important on everything.
on your own sites: try to avoid !important and use proper specificity.
